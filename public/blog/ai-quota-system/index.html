Designing Fair AI Quotas for a Fitness App

Every AI feature in Turquoise — chat, food photos, voice logs, body scans, program regeneration — costs real API tokens. Without limits, a single user could burn through a month's budget in a day. Here's how the quota system works.

Three tiers

  • Free — trial expired, not premium: all AI limits = 0
  • Trial — 7-day period with generous but bounded limits
  • Premium — doubled limits for power users

Daily general cap

Trial users get 50 general daily AI requests; Premium gets 100. Daily features (chat, food photo, voice log, macro AI) count toward this cap. When their individual limits sum above the general cap, excess is deducted from chat only — so specialized features keep their full allowance.

Reset schedules

  • Daily — midnight reset for chat, food photo, voice, macro, general
  • Weekly — Sunday reset for body scan and weekly report
  • Monthly — 1st of month for program regeneration

Resets run via a Cloudflare scheduled function (Europe/Istanbul timezone) and sync to Firestore so the app always shows accurate usage in Settings → AI Daily Quota.

80% warnings

When a user hits 80% of any limit, they get a localized warning before the hard block at 100%. Premium users never see upsell prompts — only trial users get upgrade nudges during their active trial.
